# DB Keeper — UI/UX 设计文档

> 维护说明：后续开发以 `docs/当前审查结果与后续开发计划.md` 为主依据。本文保留界面目标和交互规范参考；当前实现与本文不一致的地方，优先按主文档列出的体验问题修正。

**版本**：v1.0
**最后更新**：2026-04-14
**UI 框架**：WPF + WPF-UI (wpfui) — Windows Fluent Design

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **效率优先** | 运维工具以操作效率为核心，减少不必要的步骤和装饰 |
| **状态可见** | 连接状态、任务状态、磁盘空间等关键信息始终可见 |
| **错误防护** | 破坏性操作需二次确认，错误信息就近展示 |
| **信息层级** | 用颜色、图标、空间层次区分信息优先级 |
| **一致性** | 全局统一的颜色、间距、交互模式 |

UI需要 精心设计 若缺失素材可使用 NanoBanana自行生成素材图片

## 2. 整体布局

使用 WPF-UI 的 `NavigationView` 组件，左侧导航 + 右侧内容区：

```
┌─────────────────────────────────────────────────┐
│  DB Keeper                           — □ ×      │
├──────────┬──────────────────────────────────────┤
│          │ 页面标题              [主操作按钮]    │
│  🏠 概览  ├──────────────────────────────────────┤
│  ⏰ 任务  │                                      │
│  💾 备份  │                                      │
│  📋 日志  │         主内容区域                    │
│  ──────  │                                      │
│  🔌 连接  │                                      │
│  ⚙ 设置  │                                      │
│          │                                      │
│  ──────  │                                      │
│  连接状态 │                                      │
│  ● MES库 │                                      │
│  ● 报表库│                                      │
└──────────┴──────────────────────────────────────┘
```

### 2.1 侧边栏

| 属性 | 值 |
|------|-----|
| 组件 | WPF-UI `NavigationView` |
| 宽度 | 使用 wpfui 默认（展开约 220px，可折叠为图标模式） |
| 导航分组 | 上方：功能页面 / 下方：系统页面 |
| 底部区域 | 固定显示已配置连接的实时状态列表 |

### 2.2 顶部区域

| 属性 | 值 |
|------|-----|
| 页面标题 | 左对齐，`TitleBar` 区域 |
| 主操作按钮 | 右对齐（如「+ 新建任务」） |
| 全局预警条 | 磁盘告警时，在内容区顶部显示 `InfoBar`（橙/红色） |

|------|------|
| 正常运行 | 程序 Logo |
| 有任务执行中 | Logo + 旋转指示 |


## 5. 设计令牌

### 5.1 配色方案

基于 WPF-UI Fluent Design 系统色，自定义状态色：

| 语义 | 用途 | 色值参考 |
|------|------|----------|
| Primary | WPF-UI 主题色（跟随系统 Accent） | 系统 Accent Color |
| Success | 执行成功、连接正常、磁盘充足 | `#0F7B0F` |
| Warning | 执行警告、磁盘低、即将过期 | `#D47800` |
| Danger | 执行失败、连接断开、磁盘危险 | `#C42B1C` |
| Muted | 禁用、历史信息 | `#8E8E8E` |
| Info | 任务类型标签、普通提示 | `#0067C0` |

### 5.2 任务类型标签

| 类型 | 标签颜色 |
|------|----------|
| 备份 | Info 蓝 |
| 存储过程 | 紫色 `#744DA9` |
| 自定义 SQL | Warning 橙 |
| 清理 | Success 绿 |

### 5.3 排版

| 元素 | 说明 |
|------|------|
| 字体 | 使用 wpfui 默认（Segoe UI Variable） |
| 统计卡片数字 | 28px Bold |
| 表格内容 | 13px Regular |
| 导航项 | 14px Medium |

### 5.4 间距

| 场景 | 值 |
|------|-----|
| 页面内边距 | 24px |
| 卡片间距 | 12px |
| 卡片内边距 | 16px |
| 表格行高 | 40px |

## 6. 交互规范

### 6.1 删除确认

所有删除操作使用 wpfui `ContentDialog`：

```
┌─────────────────────────────────┐
│  ⚠ 确认删除                     │
│                                 │
│  确定要删除任务「MES 每日备份    │
│  」吗？此操作不可撤销。          │
│                                 │
│           [取消]  [确认删除]     │
└─────────────────────────────────┘
```

### 6.2 成功反馈

配置保存后使用 wpfui `Snackbar` 提示「保存成功」，3 秒自动消失。

### 6.3 耗时操作

连接测试、任务执行时，按钮显示 `ProgressRing` 加载状态。

### 6.4 空状态

列表为空时显示居中引导：

```
      📋 暂无定时任务

  创建第一个任务，开始自动化
  管理您的数据库维护工作

       [+ 新建任务]
```

### 6.5 错误展示

操作失败时使用 wpfui `InfoBar`（Severity=Error）在对应区域就近显示，不弹全屏对话框。

### 6.6 表单验证

- 输入框失焦时校验
- 错误信息显示在控件下方（红色文字）
- 提交时统一校验，自动聚焦第一个错误字段

## 7. 响应式适配

| 维度 | 要求 |
|------|------|
| 最低分辨率 | 1280×768 |
| 推荐分辨率 | 1920×1080 |

| 数据表格 | 列宽按比例，必要时水平滚动 |
