## 1. 背景与目标

### 1.1 背景

在生产环境中，SQL Server 数据库的定时备份、历史备份清理、存储过程调度等维护工作依赖人工操作或临时脚本，存在以下问题：

- 备份任务无统一管理入口，执行情况不透明
- 备份文件长期堆积，磁盘占用难以追踪
- 存储过程调度依赖 SQL Server Agent，配置复杂、权限要求高
- 维护操作无日志，问题追溯困难

### 1.2 产品定位

运行在 Windows 服务器上的**单机桌面维护工具**，用于统一管理一台或多台 SQL Server 实例的定时备份、存储过程调度和备份文件生命周期。

### 1.3 核心价值

- 可视化配置定时任务，替代手写脚本和 SQL Agent
- 备份文件全生命周期管理（创建 → 监控 → 到期清理）
- 完整的执行日志，出问题可追溯
- 磁盘空间实时监控，避免写满
- 支持多 SQL Server 实例统一管理

## 2. 约束与前提

| 项目 | 确认结果 |
|------|----------|
| 运行账户 | Windows 管理员账户 |

| 多实例管理 | 支持，可同时管理多台 SQL Server（不同 IP） |

## 3. 用户与使用场景

**目标用户**：MES 运维工程师 / DBA，熟悉 SQL Server 基本操作，不要求编程能力。

| 场景 | 描述 |
|------|------|
| 配置每日备份 | 添加「每日 02:00 全量备份 MES 库」任务，备份到 `D:\Backup`，保留 30 天 |
| 调度存储过程 | 创建「每天 23:55 执行日报汇总」任务，绑定 MES 连接，配好参数 |
| 问题排查 | 发现备份异常，查看执行日志找到错误原因 |
| 磁盘管理 | 备份盘快满了，进入备份管理页删除历史文件或调整保留天数 |
| 临时执行 | 生产异常，直接点「立即执行」手动触发一次备份 |
| 多实例管理 | 同时配置两台 SQL Server 的备份任务，统一查看日志 |

## 4. 功能模块

### 4.1 启动密码

程序启动时弹出密码输入框，校验通过后才进入主界面。密码硬编码在 `appsettings.json` 中，格式如：

```json
{
  "AppPassword": "your_password_here"
}
```

连续输错 3 次则退出程序。

**验收标准**：
- [ ] 启动程序 → 显示密码输入框，无法绕过直接进入主界面
- [ ] 输入正确密码 → 进入主界面
- [ ] 输入错误密码 → 提示错误并显示剩余次数
- [ ] 连续输错 3 次 → 程序自动退出
- [ ] 回车键等同点击「进入」按钮

---

### 4.2 连接管理

管理到多台 SQL Server 实例的连接信息。每个定时任务必须绑定一个连接。

**连接字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 连接名称 | 文本 | 是 | 界面显示用，如「MES 主库」 |
| 服务器地址 | 文本 | 是 | 如 `192.168.1.10,1433` |
| 用户名 | 文本 | 是 | SQL Server 登录名 |
| 密码 | 文本 | 是 | DPAPI 加密存储 |
| 默认数据库 | 文本 | 否 | 留空用 master |
| 连接超时 | 秒 | 否 | 默认 30 |
| 备注 | 文本 | 否 | — |

> 注：认证方式固定为 SQL Server 认证，不提供 Windows 集成认证选项。

**操作**：新建 / 测试连接 / 编辑 / 删除 / 设为默认

**连接状态**：侧边栏底部显示各连接状态（绿=正常 / 黄=延迟高 / 红=断开），每 60 秒心跳检测（`SELECT 1`）。

**验收标准**：
- [ ] 新建连接 → 填写必填字段 → 保存成功 → 列表中出现新卡片
- [ ] 必填字段为空时保存 → 提示验证错误，不允许保存
- [ ] 点击「测试连接」→ 3 秒内返回成功/失败结果
- [ ] 编辑连接 → 修改信息 → 保存 → 卡片更新显示
- [ ] 删除有关联任务的连接 → 弹出确认提示，显示关联任务数量
- [ ] 密码存入 SQLite 后为 DPAPI 加密字符串，非明文
- [ ] 侧边栏状态指示 → 连接正常绿点 / 断开红点，60 秒自动刷新

### 4.3 定时任务管理

#### 4.3.1 任务类型

| 类型 | 说明 |
|------|------|
| 备份数据库 | 执行 `BACKUP DATABASE` |
| 执行存储过程 | 调用存储过程，支持传参 |
| 执行自定义 SQL | 任意 T-SQL（DBCC、重建索引等） |
| 清理备份文件 | 按保留天数删除过期备份 |

#### 4.3.2 调度周期

| 类型 | 示例 |
|------|------|
| 每天 | 每天 02:00 |
| 每周 | 每周一 06:00 |
| 每月 | 每月 1 日 03:00 |
| 间隔 | 每 4 小时 |
| Cron | `0 0 2 * * ?` |

#### 4.3.3 任务列表

显示：任务名称、类型、绑定连接、执行周期、上次执行（时间+状态）、下次执行、启用开关。

每行操作：**立即执行** / 编辑 / 删除 / 查看日志。

#### 4.3.4 备份任务配置

| 配置项 | 说明 |
|--------|------|
| 数据库名 | 下拉选择（从连接自动获取库列表） |
| 备份类型 | 全量 / 差异 / 日志 |
| 备份目录 | 本地路径选择（文件夹浏览器） |
| 文件名规则 | `{DB}`、`{DATE}`、`{TIME}` 变量，默认 `{DB}_{DATE}.bak` |
| 保留天数 | 超期自动清理，0 = 永不删除 |
| 压缩备份 | 开关（SQL Server 2016+ 均支持） |
| 完成后校验 | 开关（`RESTORE VERIFYONLY`） |

#### 4.3.5 存储过程任务配置

| 配置项 | 说明 |
|--------|------|
| 目标数据库 | 下拉 |
| 存储过程名 | 如 `dbo.DailyReport` |
| 参数列表 | 键值对，可多行添加 |
| 执行超时 | 默认 300 秒 |

#### 4.3.6 自定义 SQL 任务配置

| 配置项 | 说明 |
|--------|------|
| 目标数据库 | 下拉 |
| SQL 内容 | 多行编辑器，支持 GO 分隔 |
| 执行超时 | 默认 600 秒 |

#### 4.3.7 立即执行

点击 → 确认对话框 → 执行 → 结果写入日志，不影响定时计划。

#### 4.3.8 启用/禁用

禁用任务不参与调度，界面灰显，操作立即生效。

### 4.3.9 执行保护

备份任务执行前进行预检查，避免无效操作和磁盘写满：

| 检查项 | 逻辑 |
|--------|------|
| 数据库大小估算 | 通过 `sp_spaceused` 获取数据库大小 |
| 磁盘剩余空间 | 检查备份目录所在磁盘剩余空间 |
| 空间判定 | 剩余空间 < 数据库大小 × 1.2 → 跳过执行 |

空间不足时记录 WARNING 日志（含数据库大小与磁盘剩余），不执行备份，等待历史备份自然过期清理后下次调度重试。任务执行超时后终止，记录 FAILED 日志，不自动重试。

**验收标准（定时任务整体）**：
- [ ] 创建备份任务 → 填写配置 → 保存 → 任务列表出现新行 → 下次执行时间正确计算
- [ ] 数据库下拉 → 自动从连接获取库列表，无需手动输入
- [ ] 配置「每天 02:00」 → 到时间自动触发 → execution_logs 新增记录 → 备份文件实际生成
- [ ] 点击「立即执行」→ 确认对话框 → 执行 → 日志记录触发方式为 MANUAL
- [ ] 禁用任务 → 到调度时间不触发 → 重新启用 → 恢复调度
- [ ] 任务执行失败 → 日志状态为 FAILED → error_detail 字段有完整错误信息
- [ ] 同时运行任务数超过并发上限 → 排队等待，不丢弃

### 4.4 备份管理

集中展示所有备份目录中的备份文件。

**文件列表字段**：文件名、数据库、大小、备份时间、过期时间、状态（正常/即将过期/已过期/手动保留）、所属任务。

**操作**：
- 手动删除 / 批量删除（二次确认）
- 手动保留（标记后自动清理跳过）
- 打开所在目录

**保留策略**（每个备份任务独立配置）：

| 策略 | 说明 |
|------|------|
| 保留天数 | 超期自动删除 |
| 最少保留份数 | 防止全删 |

**大小异常检测**：备份文件比上一份小 50% 以上时，列表标记警告。

**验收标准**：
- [ ] 备份完成 → backup_files 表新增记录 → 文件名、路径、大小正确
- [ ] 文件列表 → 能按数据库名、状态筛选 → 结果正确
- [ ] 手动删除文件 → 二次确认 → 确认后磁盘文件被删除 → 记录状态变 DELETED
- [ ] 手动保留标记 → 自动清理任务跳过该文件
- [ ] 保留天数 30 天 → 第 31 天被自动清理 → 但最少保留份数内的不删
- [ ] 文件大小比上一份小 50%+ → 列表该行显示警告标记

**文件同步**：程序每 30 分钟扫描已配置的备份目录，比对 `backup_files` 表与磁盘实际文件。表中有记录但磁盘无文件 → 更新状态为 DELETED，记录同步日志。磁盘有文件但表中无记录 → 不自动注册。同步结果写入 execution_logs（触发方式标记为 SYSTEM）。

---

### 4.5 执行日志

每次任务执行生成一条日志：任务名（快照）、类型、触发方式（定时/手动）、开始/结束时间、耗时、状态（成功/失败/警告）、输出摘要、错误详情。

**功能**：
- 倒序分页（每页 50 条）
- 按任务名、状态、时间范围过滤
- 点击展开查看完整错误
- 失败行高亮
- 导出 CSV

**保留**：默认 90 天，可配 30/60/90/180，启动时自动清理。

**验收标准**：
- [ ] 任务执行后 → 日志列表刷新 → 最新记录在最上方
- [ ] 按状态过滤「失败」→ 仅显示失败记录
- [ ] 按时间范围过滤 → 结果在范围内
- [ ] 点击失败行 → 展开显示完整错误信息
- [ ] 导出 CSV → 文件可用 Excel 打开 → 字段完整
- [ ] 设置保留 30 天 → 重启程序 → 超过 30 天的日志被清理

---

### 4.6 磁盘空间监控

自动扫描备份目录所在磁盘，显示总容量、剩余空间、使用率。

- 概览页统计卡片 + 进度条
- 低于阈值时显示警告样式
- 警告阈值：剩余 < 20%（橙色）
- 危险阈值：剩余 < 10%（红色）
- 后台每 5 分钟检测，支持手动刷新

**验收标准**：
- [ ] 配置了备份目录 → 概览页自动显示该目录所在磁盘的空间信息
- [ ] 磁盘剩余 < 20% → 概览页进度条变橙色
- [ ] 磁盘剩余 < 10% → 概览页进度条变红色
- [ ] 5 分钟后数据自动更新（无需手动操作）
- [ ] 点击「刷新」→ 立即更新磁盘数据

### 4.7 系统设置

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 关闭窗口最小化到托盘 | 开启 | 点 X 不退出 |
| 开机自启 | 关闭 | 注册表写入 |
| 日志保留天数 | 90 | 可选 30/60/90/180 |
| 任务并发上限 | 3 | 同时执行最大任务数 |
| 导出配置 | — | JSON（密码脱敏，详见下方规格） |
| 导入配置 | — | JSON，密码需手动重填 |
| 备份目录扫描间隔 | 30 分钟 | 自动同步备份文件状态 |

**导入/导出 JSON 格式**：

```json
{
  "version": "1.0",
  "exported_at": "2026-04-14T10:00:00",
  "connections": [
    {
      "name": "MES 主库",
      "host": "192.168.1.10,1433",
      "username": "sa",
      "password": "",
      "default_db": "MES_DB",
      "timeout_sec": 30,
      "remark": ""
    }
  ],
  "tasks": [
    {
      "name": "MES 每日备份",
      "task_type": "BACKUP",
      "connection_name": "MES 主库",
      "is_enabled": true,
      "schedule_type": "DAILY",
      "schedule_config": { "time": "02:00" },
      "task_config": { "database_name": "MES_DB" }
    }
  ],
  "settings": {
    "minimize_to_tray_on_close": true,
    "auto_start": false,
    "log_retention_days": 90,
    "max_concurrent_tasks": 3
  }
}
```

- 导出时：密码字段置空，任务通过 `connection_name` 关联（非 ID）
- 导入时：按 `connection_name` 匹配已有连接，密码需手动补填
| 清空日志 | — | 二次确认 |

**验收标准**：
- [ ] 开启「关闭窗口最小化到托盘」→ 点 X → 窗口隐藏，托盘图标可见，调度不停
- [ ] 开启「开机自启」→ 重启电脑 → 程序自动启动
- [ ] 导出配置 → JSON 文件中密码字段为空
- [ ] 导入配置 → 任务和连接恢复 → 密码需手动重新填写
- [ ] 清空日志 → 二次确认 → 确认后 execution_logs 表清空

## 6. 非功能性要求

| 维度 | 要求 |
|------|------|
| 启动时间 | < 5 秒 |
| 界面响应 | < 200ms |
| 调度误差 | < 30 秒 |
| 内存占用 | < 200MB |
| 日志查询 | 90 天数据 < 2 秒 |
| 操作系统 | Windows 10+ / Server 2016+ |
| SQL Server | 2016 / 2019 |
| 分辨率 | 最低 1280×768，推荐 1920×1080 |
| DPI | 100% / 125% / 150% |
| 部署 | 单文件自包含 `.exe`，免安装 .NET |

**安全**：密码 DPAPI 加密、日志不记录明文密码、导出配置密码脱敏、启动密码保护。

**可靠**：当前技术路线为 Cronos + Timer 轻量调度，SQLite 启用 WAL 模式；后续重点是补齐任务互斥、并发上限、失败追溯和导入导出可靠性。
