## 1. 背景与目标

### 1.1 背景

在生产环境中，SQL Server 数据库的定时备份、历史备份清理、存储过程调度等维护工作依赖人工操作或临时脚本，存在以下问题：

- 备份任务无统一管理入口，执行情况不透明
- 备份文件长期堆积，磁盘占用难以追踪
- 存储过程调度依赖 SQL Server Agent，配置复杂、权限要求高
- 维护操作无日志，问题追溯困难

### 1.2 产品定位

运行在 Windows 服务器上的**单机桌面维护工具**，用于统一管理一台或多台 SQL Server 实例的定时备份、存储过程调度和备份文件生命周期。

### 1.3 核心价值

- 可视化配置定时任务，替代手写脚本和 SQL Agent
- 备份文件全生命周期管理（创建 → 监控 → 到期清理）
- 完整的执行日志，出问题可追溯
- 磁盘空间实时监控，避免写满
- 支持多 SQL Server 实例统一管理

## 2. 约束与前提

| 项目 | 确认结果 |
|------|----------|
| 运行账户 | Windows 管理员账户 |
| 数据库认证 | SQL Server 认证（用户名+密码），不使用 Windows 集成认证 |
| 备份目录 | 仅本地磁盘路径，不考虑 UNC 网络路径 |
| SQL Server 版本 | 2016 / 2019（均支持备份压缩） |
| 多实例管理 | 支持，可同时管理多台 SQL Server（不同 IP） |
| 启动密码 | 单机内部工具的基础进入口令，不作为加密体系建设重点 |

## 3. 用户与使用场景

**目标用户**：MES 运维工程师 / DBA，熟悉 SQL Server 基本操作，不要求编程能力。

| 场景 | 描述 |
|------|------|
| 配置每日备份 | 添加「每日 02:00 全量备份 MES 库」任务，备份到 `D:\Backup`，保留 30 天 |
| 调度存储过程 | 创建「每天 23:55 执行日报汇总」任务，绑定 MES 连接，配好参数 |
| 问题排查 | 发现备份异常，查看执行日志找到错误原因 |
| 磁盘管理 | 备份盘快满了，进入备份管理页删除历史文件或调整保留天数 |
| 临时执行 | 生产异常，直接点「立即执行」手动触发一次备份 |
| 多实例管理 | 同时配置两台 SQL Server 的备份任务，统一查看日志 |

## 4. 功能模块

### 4.1 启动密码

程序启动时弹出密码输入框，校验通过后才进入主界面。密码硬编码在 `appsettings.json` 中，格式如：

```json
{
  "AppPassword": "your_password_here"
}
```

连续输错 3 次则退出程序。

**验收标准**：
- [ ] 启动程序 → 显示密码输入框，无法绕过直接进入主界面
- [ ] 输入正确密码 → 进入主界面
- [ ] 输入错误密码 → 提示错误并显示剩余次数
- [ ] 连续输错 3 次 → 程序自动退出
- [ ] 回车键等同点击「进入」按钮

### 4.2 连接管理

管理到多台 SQL Server 实例的连接信息。每个定时任务必须绑定一个连接。

**连接字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 连接名称 | 文本 | 是 | 界面显示用，如「MES 主库」 |
| 服务器地址 | 文本 | 是 | 如 `192.168.1.10,1433` |
| 用户名 | 文本 | 是 | SQL Server 登录名 |
| 密码 | 文本 | 是 | DPAPI 加密存储 |
| 默认数据库 | 文本 | 否 | 留空用 master |
| 连接超时 | 秒 | 否 | 默认 30 |
| 备注 | 文本 | 否 | — |

> 注：认证方式固定为 SQL Server 认证，不提供 Windows 集成认证选项。

**操作**：新建 / 测试连接 / 编辑 / 删除 / 设为默认

**连接状态**：侧边栏底部显示各连接状态（绿=正常 / 黄=延迟高 / 红=断开），每 60 秒心跳检测（`SELECT 1`）。

**验收标准**：
- [ ] 新建连接 → 填写必填字段 → 保存成功 → 列表中出现新卡片
- [ ] 必填字段为空时保存 → 提示验证错误，不允许保存
- [ ] 点击「测试连接」→ 3 秒内返回成功/失败结果
- [ ] 编辑连接 → 修改信息 → 保存 → 卡片更新显示
- [ ] 删除有关联任务的连接 → 弹出确认提示，显示关联任务数量
- [ ] 密码存入 SQLite 后为 DPAPI 加密字符串，非明文
- [ ] 侧边栏状态指示 → 连接正常绿点 / 断开红点，60 秒自动刷新

### 4.3 定时任务管理

#### 4.3.1 任务类型

| 类型 | 说明 |
|------|------|
| 备份数据库 | 执行 `BACKUP DATABASE` |
| 执行存储过程 | 调用存储过程，支持传参 |
| 执行自定义 SQL | 任意 T-SQL（DBCC、重建索引等） |
| 清理备份文件 | 按保留天数删除过期备份 |
| 数据归档 | 按时间字段将生产库单表历史数据分批迁移到历史库 |
| 目录同步 | 将照片、报表、附件等本地目录定时同步或压缩归档到备份目录 |

#### 4.3.2 调度周期

| 类型 | 示例 |
|------|------|
| 每天 | 每天 02:00 |
| 每周 | 每周一 06:00 |
| 每月 | 每月 1 日 03:00 |
| 间隔 | 每 4 小时 |
| Cron | `0 0 2 * * ?` |

#### 4.3.3 任务列表

显示：任务名称、类型、绑定连接、执行周期、上次执行（时间+状态）、下次执行、启用开关。

每行操作：**立即执行** / 编辑 / 删除 / 查看日志。

#### 4.3.4 备份任务配置

| 配置项 | 说明 |
|--------|------|
| 数据库名 | 下拉选择（从连接自动获取库列表） |
| 备份类型 | 全量 / 差异 / 日志 |
| 备份目录 | 本地路径选择（文件夹浏览器） |
| 文件名规则 | `{DB}`、`{DATE}`、`{TIME}` 变量，默认 `{DB}_{DATE}.bak` |
| 保留天数 | 超期自动清理，0 = 永不删除 |
| 压缩备份 | 开关（SQL Server 2016+ 均支持） |
| 完成后校验 | 开关（`RESTORE VERIFYONLY`） |

#### 4.3.5 存储过程任务配置

| 配置项 | 说明 |
|--------|------|
| 目标数据库 | 下拉 |
| 存储过程名 | 如 `dbo.DailyReport` |
| 参数列表 | 键值对，可多行添加 |
| 执行超时 | 默认 300 秒 |

#### 4.3.6 自定义 SQL 任务配置

| 配置项 | 说明 |
|--------|------|
| 目标数据库 | 下拉 |
| SQL 内容 | 多行编辑器，支持 GO 分隔 |
| 执行超时 | 默认 600 秒 |

#### 4.3.7 目录同步任务配置

用于现场照片、报表、附件等普通文件目录的定时备份。目录同步不依赖 SQL Server 连接，可作为本地文件维护任务独立运行。

**第一版范围**：

- 支持本地源目录同步到本地目标目录。
- 支持差量同步、全量同步、压缩归档三种模式。
- 差量同步只复制新增或变更文件，不删除目标目录中多余文件。
- 全量同步会先清空目标目录再完整复制源目录。
- 压缩归档支持 ZIP 和 7Z；7Z 依赖现场机器已安装 `7z.exe`。
- 压缩质量支持快速、均衡、最小体积三档。
- 目录同步结果纳入备份管理，压缩归档文件可被定时清理任务清理。
- 第一版不做实时监听同步，不做网络账号管理，不做文件版本库。

| 配置项 | 说明 |
|--------|------|
| 源目录 | 需要备份的照片、报表或附件目录 |
| 目标目录 | 同步目标目录或压缩归档输出目录 |
| 同步模式 | 差量同步 / 全量同步 / 压缩归档 |
| 压缩格式 | ZIP / 7Z，仅压缩归档模式显示 |
| 压缩质量 | 快速 / 均衡 / 最小体积 |
| 文件名规则 | 压缩归档文件名模板，支持 `{NAME}`、`{DATE}`、`{TIME}`、`{EXT}` |
| 保留天数 | 压缩归档文件到期时间；目录同步记录默认不自动删除目标目录 |
| 最少保留份数 | 自动清理时至少保留的压缩归档数量 |
| 包含子目录 | 默认开启 |
| 覆盖变更文件 | 默认开启；仅影响差量同步 |
| 排除规则 | 分号分隔，如 `*.tmp;*.log` |
| 执行超时 | 默认 1800 秒 |

**路径保护**：

- 源目录和目标目录不能相同。
- 目标目录不能是磁盘根目录。
- 目标目录不能位于源目录内部，避免递归复制。
- 源目录不能位于目标目录内部，避免全量清空时误删源数据。
- 全量同步只允许清空配置的目标目录，不清空其父目录。

**验收标准**：

- [ ] 创建目录同步任务 → 选择源目录、目标目录、同步模式 → 保存成功
- [ ] 差量同步首次执行 → 目标目录获得源目录全部文件和子目录结构
- [ ] 差量同步二次执行 → 只复制新增或变更文件，目标多余文件不会被删除
- [ ] 全量同步执行 → 目标目录旧内容被清空，再复制源目录当前内容
- [ ] 压缩归档选择 ZIP → 目标目录生成 `.zip` 文件，备份管理出现记录
- [ ] 压缩归档选择 7Z 但未安装 `7z.exe` → 任务失败并记录明确错误
- [ ] 目标目录等于磁盘根目录或与源目录互相嵌套 → 保存或执行时拒绝并提示
- [ ] 文件被占用无法读取 → 其他文件继续处理，任务状态为 WARNING，日志记录失败数量
- [ ] 目录同步或压缩归档结果 → 可在备份管理中查看、打开目录、手动删除或置顶

#### 4.3.8 数据归档任务配置

用于解决生产库与历史库分离场景，例如生产表只保留最近 3 个月数据，超过保留周期的数据定时迁移到历史库。

**第一版范围**：

- 支持同一 SQL Server 实例内的 `生产库.架构.表 → 历史库.架构.表`
- 支持单表归档，不处理多表级联关系
- 源表与历史表要求结构兼容，不自动创建历史表
- 按一个时间字段筛选待归档数据
- 按一个主键字段分批迁移，第一版不支持复合主键
- 支持迁移后删除源数据，也支持只复制不删除
- 不支持跨 SQL Server 实例迁移，跨实例后续可通过 Linked Server 或双连接复制扩展

| 配置项 | 说明 |
|--------|------|
| 源数据库 | 生产库，如 `MES_DB` |
| 源架构 | 默认 `dbo` |
| 源表 | 需要归档的生产表 |
| 历史数据库 | 历史库，如 `MES_History` |
| 历史架构 | 默认 `dbo` |
| 历史表 | 默认同源表，也可自定义 |
| 时间字段 | 如 `CreateTime`，用于判断哪些数据超过保留周期 |
| 主键字段 | 如 `Id`，用于分批、去重和删除 |
| 保留类型 | 按天 / 按月 |
| 保留值 | 如 `90` 天或 `3` 个月 |
| 每批行数 | 默认 5000，控制单批事务大小 |
| 单次最大行数 | 默认 50000，控制单次任务最长处理量 |
| 迁移后删除源数据 | 开启时，成功写入历史库后删除生产库对应行 |
| 历史库已存在则跳过 | 开启时按主键跳过目标表已存在数据，避免重复归档 |
| 执行超时 | 默认 1800 秒 |

**性能与锁表控制**：

- 不允许用一条大 SQL 一次性迁移和删除全部历史数据。
- 执行器必须按主键分批取数，每批独立事务提交。
- 候选数据读取使用小批量 `TOP (@BatchSize)`，并尽量跳过被业务占用的行，减少等待。
- 删除源数据只删除本批已处理主键，不执行大范围条件删除。
- 建议生产表建立 `(时间字段, 主键字段)` 索引，避免归档任务扫描全表。
- 日志必须记录批次数、候选行数、插入行数、跳过行数、删除行数和耗时。
- 任何批次插入失败时，不得删除该批源数据。

**验收标准**：

- [ ] 配置生产库、历史库、表名、时间字段、主键字段、保留周期后可保存任务
- [ ] 历史表不存在或字段不兼容时，任务执行失败并记录明确错误
- [ ] 待归档数据超过一批时，任务按批次迁移，不生成长事务
- [ ] 历史库已存在相同主键时，开启跳过后不会重复插入
- [ ] 开启“迁移后删除源数据”后，仅删除已成功处理的源表主键
- [ ] 关闭“迁移后删除源数据”后，只复制到历史库，不删除生产库数据
- [ ] 执行日志可看到归档行数、跳过行数、删除行数、批次数和耗时

#### 4.3.9 任务历史执行记录

任务列表每行操作列提供“历史记录”入口，用于按任务查看最近执行情况，提升备份、归档、清理等任务的追溯能力。

| 字段 | 说明 |
|------|------|
| 执行时间 | 开始时间与结束时间 |
| 触发方式 | 定时 / 手动 / 系统 |
| 状态 | 成功 / 警告 / 失败 / 执行中 |
| 耗时 | `duration_ms` 格式化展示 |
| 摘要 | 任务执行摘要，例如备份文件路径、归档行数 |
| 错误详情 | 失败时展示完整错误 |

对于备份任务，历史记录应尽量展示备份文件路径、文件状态和文件是否仍存在；对于数据归档任务，展示迁移统计。

**验收标准**：

- [ ] 任务列表操作列点击“历史记录”后打开该任务最近执行记录
- [ ] 可看到每次执行的时间、状态、耗时、摘要和错误详情
- [ ] 备份任务历史中能辅助判断备份文件是否还存在
- [ ] 数据归档任务历史中能看到迁移行数、跳过行数、删除行数

#### 4.3.10 立即执行

点击 → 确认对话框 → 执行 → 结果写入日志，不影响定时计划。

#### 4.3.11 启用/禁用

禁用任务不参与调度，界面灰显，操作立即生效。

### 4.3.12 执行保护

备份任务执行前进行预检查，避免无效操作和磁盘写满：

| 检查项 | 逻辑 |
|--------|------|
| 数据库大小估算 | 通过 `sp_spaceused` 获取数据库大小 |
| 磁盘剩余空间 | 检查备份目录所在磁盘剩余空间 |
| 空间判定 | 剩余空间 < 数据库大小 × 1.2 → 跳过执行 |

空间不足时记录 WARNING 日志（含数据库大小与磁盘剩余），不执行备份，等待历史备份自然过期清理后下次调度重试。任务执行超时后终止，记录 FAILED 日志，不自动重试。

**验收标准（定时任务整体）**：
- [ ] 创建备份任务 → 填写配置 → 保存 → 任务列表出现新行 → 下次执行时间正确计算
- [ ] 数据库下拉 → 自动从连接获取库列表，无需手动输入
- [ ] 配置「每天 02:00」 → 到时间自动触发 → execution_logs 新增记录 → 备份文件实际生成
- [ ] 点击「立即执行」→ 确认对话框 → 执行 → 日志记录触发方式为 MANUAL
- [ ] 禁用任务 → 到调度时间不触发 → 重新启用 → 恢复调度
- [ ] 任务执行失败 → 日志状态为 FAILED → error_detail 字段有完整错误信息
- [ ] 同时运行任务数超过并发上限 → 排队等待，不丢弃

### 4.4 备份管理

集中展示所有备份目录中的备份文件和目录同步结果。

**文件列表字段**：文件名、来源类型、来源名称、大小、备份时间、过期时间、状态（正常/即将过期/已过期/手动保留）、所属任务。

**操作**：
- 手动删除 / 批量删除（二次确认）
- 手动保留（标记后自动清理跳过）
- 打开所在目录

**保留策略**（每个备份任务独立配置）：

| 策略 | 说明 |
|------|------|
| 保留天数 | 超期自动删除 |
| 最少保留份数 | 防止全删 |

**大小异常检测**：备份文件比上一份小 50% 以上时，列表标记警告。

**来源类型**：

| 类型 | 说明 |
|------|------|
| 数据库 | SQL Server 全量、差异、日志备份 |
| 目录 | 目录差量同步、目录全量同步、ZIP/7Z 压缩归档 |

**验收标准**：
- [ ] 备份完成 → backup_files 表新增记录 → 文件名、路径、大小正确
- [ ] 文件列表 → 能按来源名称、来源类型、状态筛选 → 结果正确
- [ ] 手动删除文件 → 二次确认 → 确认后磁盘文件被删除 → 记录状态变 DELETED
- [ ] 手动保留标记 → 自动清理任务跳过该文件
- [ ] 保留天数 30 天 → 第 31 天被自动清理 → 但最少保留份数内的不删
- [ ] 文件大小比上一份小 50%+ → 列表该行显示警告标记
- [ ] 目录同步任务完成 → 备份管理出现目录来源记录，类型显示为 DIR_DIFF / DIR_FULL / DIR_ZIP / DIR_7Z

**文件同步**：程序每 30 分钟扫描已配置的备份目录，比对 `backup_files` 表与磁盘实际文件。表中有记录但磁盘无文件 → 更新状态为 DELETED，记录同步日志。磁盘有文件但表中无记录 → 不自动注册。同步结果写入 execution_logs（触发方式标记为 SYSTEM）。

---

### 4.5 执行日志

每次任务执行生成一条日志：任务名（快照）、类型、触发方式（定时/手动）、开始/结束时间、耗时、状态（成功/失败/警告）、输出摘要、错误详情。

**功能**：
- 倒序分页（每页 50 条）
- 按任务名、状态、时间范围过滤
- 点击展开查看完整错误
- 失败行高亮
- 导出 CSV

**保留**：默认 90 天，可配 30/60/90/180，启动时自动清理。

**验收标准**：
- [ ] 任务执行后 → 日志列表刷新 → 最新记录在最上方
- [ ] 按状态过滤「失败」→ 仅显示失败记录
- [ ] 按时间范围过滤 → 结果在范围内
- [ ] 点击失败行 → 展开显示完整错误信息
- [ ] 导出 CSV → 文件可用 Excel 打开 → 字段完整
- [ ] 设置保留 30 天 → 重启程序 → 超过 30 天的日志被清理

---

### 4.6 磁盘空间监控

自动扫描备份目录所在磁盘，显示总容量、剩余空间、使用率。

- 概览页统计卡片 + 进度条
- 低于阈值时显示警告样式
- 警告阈值：剩余 < 20%（橙色）
- 危险阈值：剩余 < 10%（红色）
- 后台每 5 分钟检测，支持手动刷新

**验收标准**：
- [ ] 配置了备份目录 → 概览页自动显示该目录所在磁盘的空间信息
- [ ] 磁盘剩余 < 20% → 概览页进度条变橙色
- [ ] 磁盘剩余 < 10% → 概览页进度条变红色
- [ ] 5 分钟后数据自动更新（无需手动操作）
- [ ] 点击「刷新」→ 立即更新磁盘数据

## 6. 非功能性要求

| 维度 | 要求 |
|------|------|
| 启动时间 | < 5 秒 |
| 界面响应 | < 200ms |
| 调度误差 | < 30 秒 |
| 内存占用 | < 200MB |
| 日志查询 | 90 天数据 < 2 秒 |
| 操作系统 | Windows 10+ / Server 2016+ |
| SQL Server | 2016 / 2019 |
| 分辨率 | 最低 1280×768，推荐 1920×1080 |
| DPI | 100% / 125% / 150% |
| 部署 | 单文件自包含 `.exe`，免安装 .NET |

**安全**：密码 DPAPI 加密、日志不记录明文密码、导出配置密码脱敏、启动密码保护。

**可靠**：当前技术路线为 Cronos + Timer 轻量调度，SQLite 启用 WAL 模式；后续重点是补齐任务互斥、并发上限、失败追溯和导入导出可靠性。
